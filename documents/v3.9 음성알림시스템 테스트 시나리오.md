# v3.9 음성 알림 시스템 테스트 시나리오

## 📋 개요

v3.9 DFPlayer Mini 음성 알림 시스템의 전체 기능을 검증하는 테스트 시나리오입니다.

---

## 🔧 테스트 환경

### 필수 장비
- [x] ESP32-S3 보드 (v3.9 통합 완료)
- [x] DFPlayer Mini (배선 완료)
- [x] 스피커 (연결 완료)
- [x] MicroSD 카드 (음성 파일 38개)
- [x] 시리얼 모니터 (115200 baud)

### 준비 확인
```bash
# 1. 컴파일
pio run

# 2. 업로드
pio run -t upload

# 3. 시리얼 모니터
pio device monitor
```

## ✅ 테스트 3: 볼륨 조절

### 목표
볼륨 증감 및 음소거 기능 확인

### 절차
```
1. Settings 화면 진입
2. 볼륨 슬라이더 확인
3. 볼륨 15 → 25로 변경
4. 테스트 음성 재생
5. 음소거 버튼 클릭
6. 음소거 해제
```

### 코드 테스트
```cpp
// 시리얼 모니터에서 수동 실행
void testVolume() {
    Serial.println("=== 볼륨 테스트 ===");
    
    // 볼륨 15
    voiceAlert.setVolume(15);
    voiceAlert.playSystem(VOICE_READY);
    delay(2000);
    
    // 볼륨 20
    voiceAlert.setVolume(20);
    voiceAlert.playSystem(VOICE_READY);
    delay(2000);
    
    // 볼륨 25
    voiceAlert.setVolume(25);
    voiceAlert.playSystem(VOICE_READY);
    delay(2000);
    
    // 음소거
    voiceAlert.mute();
    voiceAlert.playSystem(VOICE_READY);  // 들리지 않아야 함
    delay(2000);
    
    // 음소거 해제
    voiceAlert.unmute();
    voiceAlert.playSystem(VOICE_READY);  // 다시 들림
    delay(2000);
}
```

### 합격 기준
- [x] 볼륨 15 → 조용함
- [x] 볼륨 20 → 보통
- [x] 볼륨 25 → 큼
- [x] 음소거 시 소리 없음
- [x] 음소거 해제 시 소리 복구

## ✅ 테스트 5: 에러 음성 알림

### 목표
에러 발생 시 긴급 음성 알림 확인

### 절차 (시뮬레이션)
```cpp
// 테스트 코드 실행
void testErrorVoice() {
    Serial.println("=== 에러 음성 테스트 ===");
    
    // 1. 과열
    Serial.println("1. 과열 에러");
    voiceAlert.playError(VOICE_ERROR_OVERHEAT);
    delay(3000);
    
    // 2. 과전류
    Serial.println("2. 과전류 에러");
    voiceAlert.playError(VOICE_ERROR_OVERCURRENT);
    delay(3000);
    
    // 3. 진공 실패
    Serial.println("3. 진공 실패");
    voiceAlert.playError(VOICE_ERROR_VACUUM_FAIL);
    delay(3000);
    
    // 4. 비상정지
    Serial.println("4. 비상정지");
    voiceAlert.setVolume(30);  // 최대 볼륨
    voiceAlert.playError(VOICE_ERROR_EMERGENCY);
    delay(3000);
}
```

### 실제 에러 테스트
```
1. 온도 센서를 손으로 가열 (>65°C)
   → "온도가 높습니다. 즉시 점검이 필요합니다"

2. 전류 제한 강제 초과 (테스트 모드)
   → "과전류가 감지되었습니다. 시스템을 점검하세요"

3. 비상정지 버튼 클릭
   → "비상 정지되었습니다. 안전을 확인하세요" (최대 볼륨)
```

### 합격 기준
- [x] 에러 발생 즉시 음성 재생
- [x] 에러 내용 명확
- [x] 에러 시 볼륨 증가 (25/30)
- [x] 비상정지는 최대 볼륨 (30/30)
- [x] 반복 재생 (2회)

## ✅ 테스트 9: 음성 큐 시스템

### 목표
여러 음성 메시지 순차 재생 확인

### 절차
```cpp
void testVoiceQueue() {
    Serial.println("=== 음성 큐 테스트 ===");
    
    // 큐에 여러 메시지 추가
    voiceAlert.queueVoice(FOLDER_SYSTEM, VOICE_WELCOME);
    voiceAlert.queueVoice(FOLDER_GUIDE, VOICE_GUIDE_PLACE_BOX);
    voiceAlert.queueVoice(FOLDER_STATE, VOICE_STATE_VACUUM_ON);
    voiceAlert.queueVoice(FOLDER_STATE, VOICE_STATE_COMPLETE);
    
    Serial.printf("큐 크기: %d\n", voiceAlert.getQueueSize());
    
    // 자동 순차 재생 (loop()에서 자동 처리)
    delay(15000);  // 모든 재생 완료 대기
    
    Serial.printf("큐 크기: %d (비어야 함)\n", voiceAlert.getQueueSize());
}
```

### 예상 동작
```
1. "진공 제어 시스템을 시작합니다"
2. "박스를 올려주세요"
3. "진공을 생성하고 있습니다"
4. "작업이 완료되었습니다"
```

### 합격 기준

## ✅ 테스트 10: 자동 음성 ON/OFF

### 목표
자동 음성 기능 켜기/끄기 확인

### 절차
```
1. Settings → Voice Alert ON (기본값)
2. 상태 변경 → 음성 재생 확인
3. Settings → Voice Alert OFF
4. 상태 변경 → 음성 재생 안 됨 확인
5. Settings → Voice Alert ON
6. 테스트 음성 재생 확인
```

### 코드
```cpp
// OFF
voiceAlert.enableAutoVoice(false);
changeState(STATE_VACUUM_ON);  // 음성 없음

// ON
voiceAlert.enableAutoVoice(true);
changeState(STATE_VACUUM_HOLD);  // 음성 있음
```

### 합격 기준
- [x] OFF 시 자동 음성 없음
- [x] ON 시 자동 음성 있음
- [x] 수동 재생은 항상 작동
- [x] 설정 저장됨
